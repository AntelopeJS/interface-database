# Lookup

The `lookup` method performs a foreign key join operation, replacing a field in your documents with data from another table. It's a simplified alternative to the more flexible `join` method, designed for common one-to-one and one-to-many relationships.

## Parameters

- **other** : The table containing documents to join with
- **localKey** : The field in the current document(s) containing the foreign key reference
- **otherKey** : The field in the other table to match against

## Behavior

The `lookup` method replaces the local key field with the populated data:

- If `localKey` contains a **single value**, the result is a **single document** from the other table
- If `localKey` contains an **array of values**, the result is an **array of documents** from the other table
- The original field is replaced with the joined data (not added alongside)
- Type inference automatically tracks the transformation

## Examples

### One-to-One Relationship

Replace a foreign key with the referenced document:

```typescript
import { Database } from '@ajs/database/beta';

async function oneToOneExample() {
  const db = Database('blog');

  // Posts have an authorId field
  interface Post {
    id: string;
    title: string;
    authorId: string;
  }

  interface User {
    id: string;
    name: string;
    email: string;
  }

  // Replace authorId with the full author object
  const postsWithAuthors = await db
    .table<Post>('posts')
    .lookup(db.table<User>('users'), 'authorId', 'id')
    .run();

  // Result type: { id: string, title: string, authorId: User }
  for (const post of postsWithAuthors) {
    console.log(`${post.title} by ${post.authorId.name}`);
  }
}
```

### One-to-Many Relationship

Replace an array of foreign keys with an array of documents:

```typescript
async function oneToManyExample() {
  const db = Database('shop');

  interface Order {
    id: string;
    customerId: string;
    productIds: string[]; // Array of product IDs
    total: number;
  }

  interface Product {
    id: string;
    name: string;
    price: number;
  }

  // Replace productIds array with full product objects
  const ordersWithProducts = await db
    .table<Order>('orders')
    .lookup(db.table<Product>('products'), 'productIds', 'id')
    .run();

  // Result type: { id: string, customerId: string, productIds: Product[], total: number }
  for (const order of ordersWithProducts) {
    console.log(`Order ${order.id}:`);
    for (const product of order.productIds) {
      console.log(`  - ${product.name}: $${product.price}`);
    }
  }
}
```

### Combining with Filters

Lookup can be combined with other query operations:

```typescript
async function lookupWithFiltersExample() {
  const db = Database('myapp');

  // Get recent orders with customer information
  const recentOrdersWithCustomers = await db
    .table('orders')
    .filter((order) => order('created_at').gt(new Date('2023-01-01')))
    .lookup(db.table('customers'), 'customerId', 'id')
    .run();

  // Lookup from filtered table
  const ordersWithPremiumCustomers = await db
    .table('orders')
    .lookup(
      db.table('customers').filter({ tier: 'premium' }),
      'customerId',
      'id',
    )
    .run();
}
```

## Lookup vs Join

### When to Use Lookup

Use `lookup` when:
- You have a simple foreign key relationship
- You want to replace a field with referenced data
- You're working with one-to-one or one-to-many relationships
- Type safety and simplicity are priorities

### When to Use Join

Use `join` when:
- You need custom join predicates (not just equality)
- You want to keep both original and joined data
- You need outer joins, cross joins, or complex join conditions
- You want to transform the result into a custom structure

### Comparison Example

```typescript
async function comparisonExample() {
  const db = Database('myapp');

  // Using lookup (simple, type-safe)
  const withLookup = await db
    .table('orders')
    .lookup(db.table('customers'), 'customerId', 'id')
    .run();
  // Result: order.customerId is now the full customer object

  // Using join (flexible, custom output)
  const withJoin = await db
    .table('orders')
    .join(
      db.table('customers'),
      JoinType.Inner,
      (order, customer) => ({
        orderId: order('id'),
        orderTotal: order('total'),
        customerName: customer('name'),
        customerId: customer('id'),
      }),
      (order, customer) => order('customerId').eq(customer('id')),
    )
    .run();
  // Result: custom structure with selected fields from both tables
}
```
